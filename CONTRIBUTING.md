# Contributing to Smart Bookmark App

Thank you for your interest in contributing! This document provides guidelines for contributing to the project.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/yourusername/smart-bookmark-app.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test thoroughly
6. Commit with clear messages: `git commit -m "Add: feature description"`
7. Push to your fork: `git push origin feature/your-feature-name`
8. Open a Pull Request

## Code Style

- Use TypeScript for all new files
- Follow the existing code formatting
- Use meaningful variable and function names
- Add comments for complex logic
- Use Tailwind CSS for styling (no custom CSS unless necessary)

## Testing Checklist

Before submitting a PR, ensure:
- [ ] Code compiles without TypeScript errors
- [ ] All existing features still work
- [ ] New features work in both light and dark mode
- [ ] Real-time updates still function correctly
- [ ] App is responsive on mobile devices
- [ ] No console errors or warnings

## Feature Ideas

Potential features to work on:
- [ ] Search/filter bookmarks
- [ ] Organize bookmarks into folders/categories
- [ ] Import/export bookmarks
- [ ] Tags for bookmarks
- [ ] Share bookmarks with other users
- [ ] Browser extension
- [ ] Bookmark descriptions/notes
- [ ] Favorite/star important bookmarks
- [ ] Sort bookmarks by date, title, or URL

## Bug Reports

When reporting bugs, include:
1. Steps to reproduce
2. Expected behavior
3. Actual behavior
4. Screenshots (if applicable)
5. Browser and OS information

## Questions?

Open an issue with the "question" label.
